## Endpoint

<ParamField path="method" type="string" required>
  POST
</ParamField>

<ParamField path="url" type="string" required>
  /auth/register
</ParamField>

## Request

### Headers

<ParamField header="Content-Type" type="string" required>
  application/json
</ParamField>

### Body Parameters

<ParamField body="email" type="string" required>
  User's email address (max 255 characters)
  
  Must be a valid email format
</ParamField>

<ParamField body="password" type="string" required>
  User's password (8-128 characters)
  
  **Requirements:**
  - Minimum 8 characters
  - At least 1 uppercase letter
  - At least 1 lowercase letter  
  - At least 1 number
  - At least 1 special character (!@#$%^&*...)
  - Not a common password
</ParamField>

<ParamField body="name" type="string" required>
  User's full name (2-100 characters)
  
  Cannot contain special HTML characters (angle brackets, braces, square brackets)
</ParamField>

### Example Request

```bash cURL
curl -X POST https://api.ceboelha.com/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "password": "SecureP@ss123",
    "name": "João Silva"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.ceboelha.com/auth/register', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  credentials: 'include', // Important: include cookies
  body: JSON.stringify({
    email: 'user@example.com',
    password: 'SecureP@ss123',
    name: 'João Silva'
  })
});

const data = await response.json();
```

```python Python
import requests

response = requests.post(
    'https://api.ceboelha.com/auth/register',
    json={
        'email': 'user@example.com',
        'password': 'SecureP@ss123',
        'name': 'João Silva'
    }
)

data = response.json()
```

## Response

### Success Response (200)

<ResponseField name="success" type="boolean" required>
  Always `true` for successful requests
</ResponseField>

<ResponseField name="data" type="object" required>
  <ResponseField name="data.user" type="object" required>
    <ResponseField name="data.user._id" type="string">
      User's unique identifier (MongoDB ObjectId)
    </ResponseField>
    
    <ResponseField name="data.user.email" type="string">
      User's email address
    </ResponseField>
    
    <ResponseField name="data.user.name" type="string">
      User's full name
    </ResponseField>
    
    <ResponseField name="data.user.role" type="string">
      User's role (default: "user")
    </ResponseField>
    
    <ResponseField name="data.user.status" type="string">
      Account status (default: "active")
    </ResponseField>
    
    <ResponseField name="data.user.preferences" type="object">
      User preferences (theme, notifications, language, etc.)
    </ResponseField>
    
    <ResponseField name="data.user.stats" type="object">
      User statistics (all start at 0 for new users)
    </ResponseField>
    
    <ResponseField name="data.user.createdAt" type="string">
      ISO 8601 timestamp of account creation
    </ResponseField>
    
    <ResponseField name="data.user.updatedAt" type="string">
      ISO 8601 timestamp of last update
    </ResponseField>
  </ResponseField>
  
  <ResponseField name="data.expiresIn" type="number">
    Seconds until access token expires (typically 900 = 15 minutes)
  </ResponseField>
</ResponseField>

<ResponseField name="message" type="string" required>
  Success message: "Conta criada com sucesso! 🐰"
</ResponseField>

### Response Cookies

The following HttpOnly cookies are automatically set:

- `accessToken`: JWT access token (expires in 15 minutes)
- `refreshToken`: JWT refresh token (expires in 7 days)

## Error Responses

### Weak Password (400)

```json
{
  "success": false,
  "error": "Validation failed",
  "code": "VALIDATION_ERROR",
  "message": "Senha deve conter pelo menos uma letra maiúscula"
}
```

### Email Already Exists (409)

```json
{
  "success": false,
  "error": "Conflict",
  "code": "CONFLICT",
  "message": "Este e-mail já está cadastrado"
}
```

### Invalid Input (400)

```json
{
  "success": false,
  "error": "Validation failed",
  "code": "VALIDATION_ERROR",
  "message": "Nome deve ter entre 2 e 100 caracteres"
}
```

### Rate Limit Exceeded (429)

```json
{
  "success": false,
  "error": "Rate limit exceeded",
  "code": "RATE_LIMIT",
  "message": "Muitas requisições. Tente novamente mais tarde."
}
```

## Notes

- **Rate Limit**: 5 requests per 15 minutes per IP
- **Token Storage**: Tokens are automatically stored in HttpOnly cookies
- **Password Security**: Passwords are hashed with bcrypt before storage
- **Device Tracking**: The system tracks the device and IP address for security purposes
- **Activity Logging**: Registration events are logged for security auditing
